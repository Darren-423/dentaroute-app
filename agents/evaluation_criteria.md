# Concourse 앱 평가 기준표

Generator와 Evaluator가 공유하는 품질 기준.
의료관광 플랫폼 특성을 반영한 Concourse 전용 기준입니다.

---

## 채점 항목

### 1. 사용성 (비중: 35%)
환자와 의사가 직관적으로 사용할 수 있는가?

합격:
- 네비게이션이 명확하고 하단 탭 바에 아이콘+레이블 모두 있음
- CTA 버튼이 눈에 띄고, 활성/비활성 상태가 구분됨
- 빈 상태(empty state)에 안내 텍스트 + CTA 버튼 포함
- 금액이 무엇인지(보증금/전체/개당) 명확하게 표시
- 사용자가 현재 어디에 있고, 다음 단계가 뭔지 알 수 있음

불합격:
- 탭 바에 레이블 없이 아이콘만 있음
- 금액이 모호하게 표시됨 (특히 결제 관련)
- 빈 상태에 CTA 없이 텍스트만 있음
- 버튼 활성/비활성 구분 불가

### 2. 디자인 품질 (비중: 30%)
Concourse 디자인 시스템과 일관적이고, 의료 서비스의 신뢰감이 느껴지는가?
합격:
- Concourse 색상 토큰 사용 (환자: 보라, 의사: 틸)
- 타이포그래피 계층이 명확 (heading → body → caption)
- 여백과 간격이 일관적
- 카드, 버튼, 인풋 등이 기존 화면과 스타일 통일
- 시각적 계층(hierarchy)이 명확

불합격:
- Concourse 색상 팔레트와 다른 색상 사용
- 기존 화면과 스타일이 이질적
- 글래스모피즘을 인풋/기능 요소에 사용

### 3. 접근성 (비중: 20%)
모든 사용자가 정보를 읽고 사용할 수 있는가?

합격:
- 색상 대비 WCAG AA 이상 (4.5:1 일반, 3:1 큰 텍스트)
- 터치 타겟 44x44px 이상
- 인풋 필드 불투명 흰색 배경 + 충분한 placeholder 대비
- 보라색/틸 배경 위 흰색 텍스트의 opacity 0.85 이상

불합격:
- 서브텍스트/placeholder가 배경과 구분 안 됨
- 터치 타겟이 너무 작음
- 인풋 필드가 반투명 배경

### 4. 기능 완성도 (비중: 15%)
SPEC의 기능이 정상 구현되고, 엣지 케이스가 처리되었는가?
합격: SPEC의 모든 기능 구현, 빈/로딩/에러 상태 처리, 반응형 정상
불합격: SPEC 기능 미구현, 상태 처리 없음, 모바일 깨짐

---

## 판정

가중 점수 = (사용성×0.35) + (디자인×0.30) + (접근성×0.20) + (기능×0.15)

- 7.0 이상 → 합격
- 5.0~6.9 → 조건부 합격 (피드백 반영 후 재검수)
- 5.0 미만 → 불합격
- 사용성 또는 디자인 4점 이하 → 무조건 불합격
- 접근성에서 WCAG AA 색상 대비 미달 시 → 무조건 조건부 이상 불가

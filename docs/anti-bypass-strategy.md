# DentaRoute: Anti-Bypass (직접거래 방지) 전략

## 문제
환자와 의사가 앱 채팅에서 연락처를 교환한 뒤, 플랫폼 수수료를 피해 직접 거래하는 것.
의사가 "앱 수수료 20% 빼고 직접 하면 15% 싸게 해줄게" → 환자 입장에서 이득 → 플랫폼 매출 손실.

---

## 현재 구현된 장치들

### ✅ 1. 5% 앱 결제 할인 (당근)
- 앱으로 결제하면 치료비 5% 자동 할인
- final-payment 화면에 "🎉 5% App Payment Discount!" 배너 표시
- 플랫폼이 할인분 흡수 (의사는 원가 수령)
- **효과**: 중간. 5% vs 의사가 제안하는 15%+ 할인에는 부족

### ✅ 2. 10% 보증금 선결제 (잠금)
- 견적 선택 시 즉시 10% 보증금 결제
- 환자-의사 관계가 앱 안에서 시작
- 돈이 묶여있으므로 앱을 벗어날 이유 감소
- **효과**: 중간. 보증금은 돌려받을 수 있으므로 완전한 잠금은 아님

### ✅ 3. 티어 수수료 시스템 (의사 인센티브)
- Gold: 15%, Silver: 18%, Standard: 20%
- 앱 매출이 높을수록 수수료 감소 → 의사가 앱 안에서 거래할 동기
- 바이패스하면 티어 승급 불가
- **효과**: 장기적으로 강력. 의사가 플랫폼에 loyalty 가지게 됨

### ✅ 4. 취소 환불 정책 (이탈 비용)
- 7일+ 전: 100% 환불
- 3~7일: 50% 환불
- 3일 미만: 0% 환불
- 앱에서 빠져나가려면 보증금 일부 손실
- **효과**: 약함. 예약 초기에는 100% 환불 가능하므로 쉽게 빠져나감

## 추가 필요한 장치들

### 🔴 5. 채팅 메시지 필터링 (가장 시급)
**현재**: 채팅에서 전화번호, 이메일, 카카오톡 ID 등 자유롭게 주고받을 수 있음.
**문제**: 연락처가 오가면 앱 밖 거래는 시간문제.

**구현 방안**:
```
감지 패턴:
- 전화번호: 010-XXXX-XXXX, +82, +1 (XXX) XXX-XXXX 등
- 이메일: xxx@xxx.xxx 패턴
- 메신저: "카카오", "kakao", "카톡", "라인", "line", "whatsapp", "위챗", "telegram"
- URL: http://, www., .com, .kr 등

동작:
- 패턴 감지 시 메시지 내 해당 부분을 [contact info hidden] 으로 마스킹
- 발신자에게 경고: "For your protection, contact information is not allowed before booking confirmation."
- 예약 확정(confirmed) 후에는 필터 해제 (실제 연락 필요하므로)
```

**참고**: Airbnb가 정확히 이 방식 사용. 예약 전까지 연락처 공유 불가, 확정 후 해제.

### 🔴 6. 의사 파트너 계약 조항 (비기술적, 가장 강력)
기술적 장치보다 법적 구속력이 강함.

**계약 포함 조항**:
- "DentaRoute를 통해 연결된 환자와 플랫폼 밖에서 직접 거래 시 파트너십 해지"
- "직접 거래 발견 시 해당 거래액의 X%를 위약금으로 부과"
- "최초 연결 후 12개월간 해당 환자와의 거래는 DentaRoute 경유 필수"
- Uber, DoorDash, Upwork 등 모든 마켓플레이스가 이 방식 사용

### 🟡 7. 치료 보증 (Warranty) — 앱 결제 전용
- 앱으로 결제한 치료에 대해서만 치료 보증 적용
- "임플란트 5년 보증, 크라운 3년 보증 — DentaRoute 결제 시에만"
- 환자 입장: 직접 거래하면 문제 생겨도 보호 없음
- **매우 강력한 동기** — 치과 치료는 고가이고 실패 리스크가 있음

### 🟡 8. 리뷰 시스템 연동
- 앱 결제 완료한 환자만 리뷰 작성 가능
- 의사 입장: 리뷰가 쌓여야 새 환자 유입 → 앱 안에서 거래해야 리뷰 받음
- 환자 입장: 리뷰 있는 의사를 선호 → 리뷰 많은 의사 = 앱 활성 의사

### 🟢 9. 분할 결제 편의성 강조
- 방문별 분할 결제, 이월, 자동 인보이스 — 앱 전용 기능
- 직접 거래 시 이 편의성 없음 (현금/계좌이체로 복잡)
- 마케팅: "DentaRoute pays in installments. No hassle."

### 🟢 10. 의사 대시보드 가치 제공
- 수익 분석, 환자 관리, 자동 번역 채팅 — 의사가 앱을 떠나기 싫게
- 앱이 단순 중개가 아니라 "의사의 비즈니스 툴"이 되면 이탈 감소

| **당근** | 치료 보증 (앱 전용) | 환자 | ⭐⭐⭐⭐ |

## 구현 우선순위

1. **채팅 필터링** — 기술적으로 간단, 효과 큼, 바로 구현 가능
2. **파트너 계약 조항** — 코드 필요 없음, 계약서에 추가하면 됨
3. **치료 보증 프로그램** — UI에 표시 + 마케팅, 환자 심리에 강력
4. **나머지** — 이미 구현됐거나 점진적 추가

---

## 현실적 관점

100% 방지는 불가능해요. Airbnb도, Upwork도, 모든 마켓플레이스가 이 문제를 겪어요.
핵심은 "직접 거래보다 앱이 더 편하고, 더 안전하고, 크게 비싸지 않다"는 인식을 만드는 거예요.
기술적 벽 + 계약적 채찍 + 가치 기반 당근의 조합이 정답이에요.
